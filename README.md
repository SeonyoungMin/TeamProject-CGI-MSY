# MOGU - 신뢰 기반 중고거래 플랫폼

개인 간 중고거래에서 발생하는 사기 거래와 허위 매물 문제를 해결하기 위해 개발한 중고거래 플랫폼입니다.

실명 인증, 동네 인증, 거래 승인 트랜잭션, AI 품질 분석 기능을 통해 보다 안전한 거래 환경을 제공하는 것을 목표로 했습니다.

## 프로젝트 개요

- 개발 기간 : 2026.04 ~ 2026.06
- 개발 인원 : 2인 팀 프로젝트
- 담당 역할 : 백엔드 전반 (인증·보안, 거래 시스템, AI 연동)
- 배포 환경 : AWS EC2 + Tomcat

## 주요 기능

### 1. 계좌 실명 인증 기반 안전 거래

#### 문제

계좌이체 거래 시 판매자 본인 여부를 확인하기 어려워 사기 위험이 존재했습니다.

#### 해결

- 금융결제원 오픈뱅킹 API 연동
- 계좌번호 + 생년월일 검증
- 서버 단 검증 처리
- 타임스탬프 기반 거래번호 생성

#### 결과

수취인 실명을 확인한 후 거래를 진행할 수 있는 안전 거래 환경을 구축했습니다.

---

### 2. 거래 승인 트랜잭션

#### 문제

여러 사용자가 동시에 동일 상품을 구매 요청할 경우 중복 거래가 발생할 수 있었습니다.

#### 해결

- `@Transactional` 적용
- 주문 승인
- 상품 상태 변경
- 다른 주문 자동 취소

위 과정을 하나의 트랜잭션으로 처리했습니다.

#### 결과

동시 요청 상황에서도 데이터 정합성을 유지할 수 있도록 구현했습니다.

---

### 3. AI 상품 분석

#### 문제

중고거래 상품 설명의 신뢰도를 사용자가 직접 판단해야 했습니다.

#### 해결

- Gemini API 이미지 분석
- 상품 상태 평가
- 도용 가능성 분석
- 네이버 쇼핑 시세 비교

#### 결과

구매자가 객관적인 정보를 기반으로 거래 여부를 판단할 수 있도록 구현했습니다.

## 트러블슈팅

### AWS 배포 후 이미지가 사라지는 문제

#### 문제

로컬 파일 시스템에 저장된 이미지가 서버 재시작 시 사라지는 문제가 발생했습니다.

#### 원인

배포 서버 환경에서 로컬 파일 시스템에 의존하는 구조였기 때문입니다.

#### 해결

- imgbb API 도입
- 이미지 업로드 후 URL 발급
- DB에는 URL만 저장하도록 구조 변경

#### 결과

배포 환경과 개발 환경 모두에서 동일한 이미지를 안정적으로 제공할 수 있게 되었습니다.
